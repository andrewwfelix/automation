## 2. Technical Advice (Prompts, Workflow, Output)

### Prompting Best Practices
Set a clear system message like "You are an expert critic specializing in book vs. film comparisons; output in HTML format with precise structure." Adjust temperature to 0.7 for creative yet consistent analysis, and specify output format as raw markdown or HTML to match your site's template. Use structured prompts: "Analyze [Book Title] vs. [Film Title], covering plot fidelity, character changes, and a verdict—include affiliate links naturally."

### Known Refusals & Workarounds
Grok may refuse explicit affiliate link insertions if they seem promotional; workaround by framing as "suggested reading resources" and embedding them post-generation. For HTML structure, if it outputs plain text, use conditional prompts like "If generating code, wrap in <div> tags." It might balk at overly opinionated tones; mitigate with "Maintain a balanced critic voice with slight bias toward books for engagement."

### Token Efficiency & Cost per Page
Aim for prompts under 2,000 tokens by summarizing inputs (e.g., feed key book/film details instead of full texts); this keeps costs low, estimating $0.10–$0.20 per page via xAI API. Batch similar comparisons to reuse context, reducing redundant tokens.

### Integration Patterns
Integrate via xAI API for direct calls, stripping unnecessary metadata from outputs before merging into your CMS. For hybrid workflows, generate base content with Grok, then merge with manual edits or other tools; automate by scripting API responses to populate page templates, ensuring comparison panels are preserved.

### Known Issues to Watch
Grok can occasionally over-emphasize humor in critiques, diluting analysis—add prompts to "focus on factual comparisons." Watch for context window limits (128k tokens); split long inputs. Experimental nature means varying refusal patterns, so test prompts iteratively and have fallbacks like rephrasing for sensitive topics.
